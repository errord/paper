# §2.5 网络时变性与动态配置框架

## 2.5.1 资产相关性的时变特征

金融资产的相关性结构并非静态不变。大量实证研究表明：

1. **相关性崩溃现象**（Longin & Solnik, 2001）：资产相关性在市场压力时期显著上升，在平稳期回落。这意味着基于单一静态窗口构建的网络可能低估或高估资产间的联动程度。

2. **结构性变化**（Engle, 2002）：在不同货币政策周期和经济状态中，相关性结构呈现根本性变化——行业轮动、政策转向、宏观冲击等都可能引发网络拓扑的质变。

3. **核心-外围角色的动态转换**：某些在平稳期位于网络外围的资产，可能在危机期间因流动性枯竭或恐慌传染而跃升为网络核心。

使用固定时间窗口构建单一静态网络，隐含了"相关结构稳定"的强假设，可能导致以下问题：

- 在市场剧变时期，网络拓扑已发生本质变化，但配置策略仍基于过时的网络结构；
- 社区成员的动态调整被忽视，某些资产可能在不同时期属于不同社区；
- 中心性排序的变化未被捕捉，原先的外围资产可能因结构性变化成为新的风险传导核心。

## 2.5.2 滚动窗口网络构建

为捕捉网络的时变特征，本文采用滚动窗口方法动态更新网络结构。

### 参数设置

| 参数 | 符号 | 取值 | 含义 |
|------|------|------|------|
| 滚动窗口长度 | $\tau$ | 252个交易日（约一年） | 估计相关矩阵的数据量 |
| 调仓步长 | $\Delta$ | 21个交易日（约一个月） | 权重更新频率 |

在时点 $t$ 的网络 $G_t$ 基于 $[t-\tau, t]$ 的数据构建。

### 窗口长度的选择依据

窗口长度 $\tau$ 面临估计精度与时变性之间的经典权衡：

$$
\text{估计精度} \propto \sqrt{T} \quad \leftrightarrow \quad \text{时变捕捉能力} \propto 1/T
$$

- 窗口过短（如 $\tau = 63$，一个季度）：相关矩阵估计噪音过大（$n/T = 30/63 \approx 0.48$），网络结构剧烈波动，权重频繁大幅调整，换手成本高昂；
- 窗口过长（如 $\tau = 504$，两年）：网络过度平滑，无法捕捉结构性变化，在市场转折点反应迟缓；
- $\tau = 252$（一年）：$n/T = 30/252 \approx 0.12$，Marchenko-Pastur上界 $\lambda_+ \approx 1.69$，能分离出约4-6个信号特征值，在估计精度与时变性之间取得平衡。

**稳健性检验。** 本文对 $\tau \in \{126, 189, 252, 378, 504\}$ 进行敏感性分析（§2.6.3(2)），验证结论对窗口选择的依赖程度。

## 2.5.3 网络稳定性度量

为监测网络结构的变化并决定是否需要调仓，定义以下网络稳定性指标：

### 指标一：边重叠率（Jaccard Edge Overlap）

$$
J(G_t, G_{t-\Delta}) = \frac{|E_t \cap E_{t-\Delta}|}{|E_t \cup E_{t-\Delta}|}
$$

即相邻时间窗口网络的Jaccard相似系数。$J$ 越高表示网络边结构越稳定。

**解释标准：**
- $J > 0.8$：网络高度稳定，无需调仓；
- $0.5 < J \leq 0.8$：网络中度变化，正常调仓；
- $J \leq 0.5$：网络剧烈变化，需紧急重构。

### 指标二：中心性秩相关（Centrality Rank Correlation）

$$
\rho_s(t) = \text{Spearman}\left(\{C_B^{(t)}(i)\}_{i=1}^n, \; \{C_B^{(t-\Delta)}(i)\}_{i=1}^n\right)
$$

即相邻时间窗口中心性排序的Spearman秩相关系数。若 $\rho_s(t)$ 显著下降，表明风险传导结构发生质变。

**解释标准：**
- $\rho_s > 0.8$：中心性排序高度稳定；
- $\rho_s \leq 0.5$：核心-外围关系发生根本重组。

### 指标三：社区稳定性（Normalized Mutual Information）

$$
NMI(t) = \frac{2I(\mathcal{C}_t; \mathcal{C}_{t-\Delta})}{H(\mathcal{C}_t) + H(\mathcal{C}_{t-\Delta})}
$$

其中 $I(\cdot;\cdot)$ 为互信息，$H(\cdot)$ 为熵。NMI衡量相邻时间窗口社区划分的一致性。

**解释标准：**
- $NMI > 0.8$：社区结构高度稳定，跨社区分散化方案无需调整；
- $NMI \leq 0.5$：社区成员发生大规模重组，需要重新检测社区结构。

## 2.5.4 自适应调仓规则

基于上述网络稳定性指标，本文设计了自适应调仓机制，以在降低换手成本与及时响应网络变化之间取得平衡：

### 调仓触发条件

在每个潜在调仓时点 $t$，计算稳定性指标 $J(G_t, G_{t-\Delta})$ 和 $NMI(t)$：

$$
\text{触发调仓} \iff J(G_t, G_{t-\Delta}) < J_{threshold} \quad \text{或} \quad NMI(t) < NMI_{threshold}
$$

默认阈值设置：$J_{threshold} = 0.7$，$NMI_{threshold} = 0.8$。

### 调仓逻辑

```
IF 到达固定调仓周期 (每Δ=21天):
    计算 J 和 NMI
    IF J < J_threshold 或 NMI < NMI_threshold:
        重构网络 → 重新计算中心性和社区 → 重新优化权重
    ELSE:
        维持原有权重（跳过本次调仓）
```

### 理论意义

自适应调仓规则的价值在于：

1. **降低换手成本**：当网络稳定时，跳过不必要的调仓，直接减少交易成本。在交易成本为10bp的情况下，自适应调仓可降低年化换手率约30-50%。

2. **及时响应结构变化**：当网络发生质变（如危机期间社区大幅重组），能够及时触发权重调整，避免基于过时网络的配置。

3. **与融合方法的协同**：对于网络增强BL和社区BL-RP混合策略，网络结构的变化直接影响BL观点和社区分层，自适应调仓确保了这些策略输入的时效性。

## 2.5.5 滚动回测框架

整合上述动态配置元素，本文的滚动回测框架如下：

### 时间线

```
数据期: [2023-01-01, 2025-12-31]   (3年，约726个交易日)
训练窗口: τ = 252天
样本外期: [2024-01-xx, 2025-12-31]  (约474个交易日)
调仓频率: 每Δ=21个交易日
```

### 每个调仓时点的流程

```
For t = τ, τ+Δ, τ+2Δ, ...:
    1. 取训练窗口 [t-τ, t] 的收益率数据
    2. 计算 Ledoit-Wolf 协方差矩阵（所有策略共用）
    3. 构建 PMFG 网络、计算中心性、检测社区（图模型策略用）
    4. [可选] 计算 RMT 去噪协方差（去噪RP用）
    5. 对每个策略，调用其权重函数得到 w_t
    6. 计算组合收益率 r_{p,t+1} = w_t · r_{t+1}
    7. [可选] 扣除交易成本 tc = tc_bps × |w_t - w_{t-1}|_1
    8. 滚动至下一时点
```

### 绩效指标计算

基于样本外期的组合日收益率序列 $\{r_{p,t}\}$，计算以下指标：

| 指标 | 公式 | 含义 |
|------|------|------|
| 年化收益 | $\mu_{ann} = \bar{r}_p \times 252$ | 平均日收益率年化 |
| 年化波动 | $\sigma_{ann} = \sigma_p \times \sqrt{252}$ | 日收益率标准差年化 |
| Sharpe比率 | $SR = \mu_{ann} / \sigma_{ann}$ | 风险调整后收益 |
| 最大回撤 | $MDD = \max_{t_1 < t_2} \frac{V_{t_1} - V_{t_2}}{V_{t_1}}$ | 峰谷最大跌幅 |
| CVaR(95%) | $CVaR = E[r | r \leq VaR_{0.05}]$ | 尾部风险度量 |
| Calmar比率 | $Calmar = \mu_{ann} / |MDD|$ | 收益/回撤比 |
| 偏度 | $Skew = E[(r - \bar{r})^3] / \sigma^3$ | 收益分布对称性 |
| 峰度 | $Kurt = E[(r - \bar{r})^4] / \sigma^4$ | 尾部厚度 |
| 换手率 | $TO = \frac{1}{T_{reb}}\sum_t \|\mathbf{w}_t - \mathbf{w}_{t-1}\|_1$ | 平均每次调仓的权重变动 |
| 有效资产数 | $N_{eff} = 1/\sum_i w_i^2$ | Herfindahl等价数 |
